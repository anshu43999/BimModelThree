本轮验收需要特别关注：

- 模型树：四类 tabs、懒加载、默认不展开详细树、点击节点后模型对应构件高亮/定位。
- 测量：测量页面点击模型应进入选点测量，不应触发普通构件选择；需要确认捕捉点、测量线和距离文本。
- 标签：点击模型选择构件后可新增标签，标签支持编辑、调整气泡位置、删除、定位和刷新后恢复；默认只显示当前构件气泡，可手动显示全部气泡；支持深色、蓝色、警告、浅色、高对比和完整自定义气泡样式。
- 批注：状态过滤已经调整为 tabs；新增/编辑表单中的状态仍是下拉框；支持在不改变绑定构件的前提下调整气泡位置；默认只显示当前构件气泡，可手动显示全部气泡；支持与标签共用的气泡样式切换；编辑并改变状态后模型气泡不应消失，除非被当前筛选 tab 排除。
- 路径漫游：打开路径漫游面板后，可新建/切换/保存多个本地漫游方案；每个方案可把当前相机视角记录为路径点；路径点支持命名、时间编辑、上移/下移、更新当前视角、恢复和删除；至少 2 个路径点后可播放、暂停、停止，并可调整播放速度；主 Viewer 播放经过关键帧时自动触发已记录的选择/变换状态；播放期间不应与测量、框选、自由巡检、模型变换冲突。
- 视点、标签、批注当前主 Viewer 仍主要使用 `localStorage`；后端已提供 JSON 文件型业务数据基础 API、SDK adapter、主 Viewer 手动同步入口、业务上下文字段配置和业务数据模式选择，支持 `local` 本地模式、`manual` 手动同步、`backend` 后端优先。后端优先模式会启用视点、标签、批注本地变更自动同步和模型加载后自动拉取；快照支持推送、远端列表拉取、视角/选择恢复和删除，待人工联调、默认后端化、数据库化、多用户协同、权限和审计。

## 4. 通用通过标准

每个功能通过前必须满足：

1. 操作过程中页面不崩溃、不白屏。
2. Viewer 控制台无阻断性错误。
3. 操作不会破坏模型加载、相机控制、选择状态和属性面板。
4. 大模型下操作不能出现长时间无响应。
5. 涉及业务绑定时优先记录 `GlobalId`，`localId` 只作为当前运行时 ID。
6. 不能确认的数据必须记录为“待确认”，不能直接判定通过。

## 6. A001 模型加载验收

| 项 | 内容 |
|---|---|
| 验收目标 | 验证 `.frag`、URL、Manifest 三种入口是否能稳定加载模型。 |
