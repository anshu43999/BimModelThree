# BIM 模型项目进度评估与进度计划表

更新时间：2026-07-16

文档状态：当前有效，作为管理排期和剩余工作量的权威主表。

## 1. 评估口径

本文基于当前 `E:\BimModelThree` 项目代码和 `docs` 下最新文档，对 BIM 模型系统当前进度、剩余工作量和后续排期进行评估。

评估口径：

- 人力假设：`1 名前端 + 1 名后端 + AI 协作`。
- 天数单位：人日，表示从当前状态继续推进的剩余实现天数，不包含业务方长时间等待、正式评审等待和非技术沟通等待。
- 难度取值：低、中、中高、高、很高。
- 完成进度：按“可产品化交付”口径评估，不只按前端 demo 是否可演示评估。
- 当前很多能力已经完成前端基础版，但标签、批注、视点等仍主要使用 `localStorage`，所以整体进度不能按 100% 计算。

当前基线：

| 类别 | 当前结论 |
|---|---|
| 主模型格式 | `.frag` + `manifest.json` |
| 主 Viewer | `viewer-service/viewer/index.html` |
| 功能演示页 | `viewer-service/viewer/demo.html` |
| 当前 manifest 入口 | `/converted/manifest.json`，具体模型以部署产物为准 |
| 前端基础能力 | 模型加载、树/属性、选择、测量、剖切、视点、标签、批注、分区、多模型、漫游和双版本对比 |
| 前端架构收口 | ToolMode、CameraControl、ViewportPanel、ModelRegistry、语义索引、版本对比和 Runtime SDK 已模块化 |
| 自动化基线 | 2026-07-16：Viewer Node 测试 `109 passed / 0 failed` |
| 主要短板 | 真实模型人工验收、T003/T004、第三方试接入、数据库/权限/审计、页面继续解耦 |

总体判断：

- 当前项目已具备第一版交付和第三方集成基础，不再继续扩展新的 Viewer 功能。
- 前端已经覆盖主要 BIM 查看、业务标记、装配、漫游和版本对比场景，并完成一轮状态控制与 SDK 解耦。
- 后端仍以转换、Manifest、文件型业务数据 API 为主，数据库、对象存储、权限和审计明显不足。
- 后续主线是“真实模型验收 + T003/T004 + 第三方试接入 + 后端产品化 + 页面继续解耦”。

## 8. 结论

当前项目进度可以判断为：**Viewer 前端功能与第一轮架构收口已基本完成，前端开发完成度约 85%-90%，整体产品化完成度约 70%-75%；真实模型验收和后端产品化是主要短板**。

近期最合理的工作重点不是继续扩展复杂功能，而是：

- 用真实模型把当前能力验收清楚。
- 把 `localStorage` 里的视点、标签、批注迁移到后端接口。
- 用真实第三方系统验证 Manifest、SDK、Origin 和资源授权。
- 继续将页面逻辑迁移到 Controller、Engine 和 SDK，控制维护成本。
