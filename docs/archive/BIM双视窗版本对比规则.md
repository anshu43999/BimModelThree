# BIM 双视窗版本对比规则

更新时间：2026-07-16

文档状态：当前有效的专项规则；实现已覆盖 GlobalId、属性和几何差异基础版，仍待真实双版本模型验收。

## 1. 文档目的

本文档沉淀当前 Viewer 双视窗版本对比的识别规则、标记规则、交互规则和已知边界，作为后续验收、后端对比服务设计和产品说明的依据。

当前实现为前端基础版，入口为：

- `viewer-service/viewer/index.html`
- `viewer-service/viewer/main-mvp.js`
- `viewer-service/viewer/engines/render-engine.js`
- `viewer-service/viewer/engines/interaction-engine.js`

## 2. 基础原则

1. 左侧视窗加载主版本模型，右侧视窗加载对比版本模型。
2. 跨版本构件匹配优先使用 IFC `GlobalId`。
3. `localId` 只作为当前模型加载会话内的运行时操作 ID，不作为跨版本业务主键。
4. 无 `GlobalId` 的构件不能做可靠跨版本匹配，只计入无 ID 统计。
5. 当前几何变更检测采用轻量包围盒比较，不做网格级布尔差异。

## 3. 差异类型

| 差异类型 | 判断规则 | 标记位置 | 标记颜色 |
|---|---|---|---|
| 相同 | 左右两侧存在相同 `GlobalId`，且未检测到属性或几何变化 | 不额外标记 | 无 |
| 左缺失 / 删除 | 左侧存在 `GlobalId`，右侧不存在同 `GlobalId` | 左侧主版本 | 红色 |
| 右新增 | 右侧存在 `GlobalId`，左侧不存在同 `GlobalId` | 右侧对比版本 | 蓝色 |
| 属性变更 | 左右存在同 `GlobalId`，但关键属性或扁平化属性值不同 | 左右两侧 | 橙色 |
| 几何变更 | 左右存在同 `GlobalId`，但包围盒中心、尺寸或体积超过阈值 | 左右两侧 | 橙色 |
| 无 ID | 构件无法读取 `GlobalId` | 只进入统计 | 不标记 |

## 4. 属性变更规则

属性比较分两层：

1. 优先比较常用业务字段：
   - `Name`
   - `entityName`
   - `Category`
   - `ObjectType`
   - `PredefinedType`
   - `Tag`
   - `Description`
2. 再比较扁平化后的属性集合。

以下身份字段不参与属性变更判断，避免误报：

- `GlobalId`
- `guid`
- `localId`
- `expressId`
- `modelId`

当前前端列表最多记录前 12 个属性差异，用于展示和定位，不作为完整审计报告。

## 5. 几何变更规则

当前几何比较使用构件包围盒，不直接比较三角面、顶点或布尔差异。

比较指标：

- 包围盒中心点偏移
- 包围盒尺寸变化
- 包围盒体积变化

当前阈值：

- 绝对阈值：`0.001`
- 相对阈值：`0.5%`

只要中心点、尺寸或体积任一项超过阈值，即标记为几何变更。

该规则适合前端快速筛查，不适合作为工程结算级或审计级几何差异结论。后续如需要高精度几何对比，应在后端使用转换产物索引、构件 mesh hash、顶点摘要或专业几何引擎补充。

## 6. 颜色规则

| 状态 | 颜色 | 说明 |
|---|---|---|
| 普通选中 | 绿色 | 左侧选中与右侧联动选中使用同一套高亮材质 |
| 删除 / 左缺失 | 红色 | 左侧存在、右侧不存在 |
| 新增 / 右新增 | 蓝色 | 右侧存在、左侧不存在 |
| 属性或几何变更 | 橙色 | 左右都有同 `GlobalId`，但内容发生变化 |
| 差异列表定位焦点 | 黄色 | 点击差异列表中的条目后临时强调 |

## 7. 交互规则

### 7.1 视口联动

- 左侧主视窗控制相机。
- 开启联动后，右侧视窗跟随左侧相机。
- 当用户从差异列表定位右侧新增构件时，会关闭联动，避免右侧定位被左侧相机覆盖。

### 7.2 选择联动

- 用户点击左侧构件后，系统读取左侧构件 `GlobalId`。
- 右侧通过同 `GlobalId` 回查当前版本 `localId`。
- 找到后右侧同步绿色高亮。
- 单选时，右侧会同步定位到对应构件。
- 多选或框选时，右侧最多同步前 200 个匹配构件，副视窗只按主选构件定位一次。

### 7.3 差异列表定位

| 列表类型 | 点击定位行为 |
|---|---|
| 左缺失 | 定位并选中左侧构件 |
| 右新增 | 定位并高亮右侧构件 |
| 变更 | 定位左侧构件，并同步定位右侧同 `GlobalId` 构件 |

## 8. 扫描范围与性能边界

当前前端基础版为避免大模型卡顿，存在扫描限制：

| 项目 | 当前限制 |
|---|---:|
| `GlobalId` 新增/删除扫描 | 前 20000 个构件 |
| 属性/几何变更扫描 | 前 1200 个共同构件 |
| 右侧选择联动 | 前 200 个选中构件 |
| 高亮上限 | 前 5000 个差异构件 |

如果模型较大，统计结果会显示扫描限制提示。正式产品化建议将完整差异计算迁移到后端或转换阶段生成索引。

## 10. 前端补全规则

### 10.1 高亮优先级

当前前端存在多类高亮状态，优先级从高到低为：

1. 差异列表定位焦点：黄色，仅用于用户点击差异项后的临时定位。
2. 普通构件选中：绿色，用于左侧选择和右侧联动选择。
3. 版本差异结果：红色、蓝色、橙色，分别表示左缺失、右新增、属性/几何变更。

当用户重新选择构件时，选中构件以绿色优先展示；取消或切换选择后，未被选中的差异构件恢复原差异颜色。

### 10.2 对比重跑与状态清理

每次点击“对比”前会清除上一轮差异高亮和差异列表状态，避免旧结果叠加到新结果上。右侧版本被清除或重新加载时，会取消当前未完成的对比任务，旧任务不允许回写到新模型。

### 10.3 列表筛选与详情

差异列表提供以下筛选项：

- 全部
- 变更
- 左缺失
- 右新增
- 无 ID

“无 ID”构件当前只做数量统计，不做定位。变更项支持展开详情，展示前端识别到的属性差异字段、左右值，以及包围盒中心、尺寸、体积差异。

## 11. 后续建议

1. 将完整差异计算后端化，避免大模型前端扫描卡顿。
2. 在转换阶段生成 `GlobalId -> localId`、属性摘要、几何摘要索引。
3. 增加导出差异报告：JSON、Excel、Markdown。
4. 后续如做高精度几何差异，建议使用构件 mesh hash、顶点摘要或后端几何引擎，不建议完全依赖前端实时计算。
5. 如需右侧独立选择，应补充右侧视口射线拾取和双向选择同步策略。
