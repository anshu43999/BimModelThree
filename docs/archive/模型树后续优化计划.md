## 背景

当前模型树已经完成最小 MVP 所需的真实 `.frag` 数据对齐：

- `models` 区分几何构件、空间树实体和全量 IFC 实体。
- `objects` 基于真实 `getSpatialStructure()`，并跳过无意义 bridge 节点。
- `classes` 默认展示可交互几何类别，非几何实体折叠展示。
- `storeys` 基于真实楼层空间结构统计可选择构件。
- 树节点名称列与数量/状态 meta 已拆分。
- 具体构件名称通过异步读取 `entityName / Name / LongName` 补齐。

后续优化主要围绕大模型交互性能、树节点详情表达、搜索过滤和属性读取效率展开。

## P1：大分组选择性能优化

### 问题

当用户点击 `IFCBEAM`、`IFCSLAB`、`geometric elements` 等大分组节点时，可能一次选择并高亮数千到数万个构件。即使已经减少了全量 GUID 解析，`model.highlight()` 本身仍需要处理大量构件，会出现卡顿。

### 建议方案

- 增加大分组高亮阈值，例如 `500` 或 `1000`。
- 小于阈值时直接高亮。
- 大于阈值时只选中树节点并显示分组摘要，不立即高亮全部。
- 在右侧面板提供明确操作：
  - `高亮全部`
  - `隔离全部`
  - `展开查看构件`
- 高亮全部操作需要显示处理中状态，并允许后续扩展取消操作。

### 验收标准

- 点击大分组节点时 UI 不明显卡顿。
- 用户能明确知道当前选中了哪个分组、包含多少构件。
- 用户可以主动触发高亮全部，而不是默认立刻执行高成本操作。

## P2：树节点详情面板区分分组与构件

当前状态：已实现基础版。点击分组节点时显示分组摘要，并提供“高亮全部 / 隔离全部 / 展开查看”；点击叶子构件节点时才显示真实 BIM 属性。

### 问题

当前点击分组节点时，右侧属性面板仍可能使用 `primaryLocalId` 的构件属性作为展示内容，容易让用户误以为分组本身就是该构件。

### 建议方案

- 引入树节点类型：
  - `model-summary`
  - `class-group`
  - `storey-group`
  - `spatial-group`
  - `element`
  - `metadata-group`
- 点击分组节点时显示分组摘要：
  - 节点类型
  - IFC 类别或分组名称
  - 可选择构件数
  - 子节点数
  - 是否可直接高亮
- 点击具体构件节点时才显示真实 BIM 属性。

### 验收标准

- 分组节点不显示某个子构件的 `entityName`、`GlobalId` 等属性。
- 构件节点保持完整属性展示。

## P2：模型树搜索与过滤

### 问题

真实模型构件数量较大，仅靠展开树难以快速定位目标。

### 建议方案

支持按以下字段搜索：

- `entityName / Name / LongName`
- `IFC class`
- `localId`
- `GlobalId`

过滤方式：

- 高亮匹配节点。
- 可选择只显示匹配路径。
- 支持清空搜索恢复完整树。

### 验收标准

- 用户可以在大模型中快速定位具体构件或类别。
- 搜索不阻塞主渲染线程。

## P2：属性读取批量化与缓存

### 问题

当前可见树节点名称通过逐个异步读取属性补齐。大量展开时会产生多次 worker 查询，可能导致 UI 抖动或等待时间过长。

### 建议方案

- 对可见节点进行分批读取。
- 缓存 `localId -> displayName`。
- 对已读取失败的节点缓存失败状态，避免重复查询。
- 后续可扩展批量 `getItemsData(localIds)`。

### 验收标准

- 展开大分组时不会一次发起过多属性查询。
- 已读取过的节点再次出现时立即显示名称。

## P3：非几何实体详情浏览

当前状态：已实现基础版。`all IFC entities`、`non-geometric entities` 及其子类节点会保留 `dataLocalIds`，点击后展示数据节点摘要和样例实体；这类节点不触发模型选择、高亮或隔离。

### 问题

当前非几何实体被折叠到 `non-geometric entities`，主要用于避免误高亮。但后续如果要做 BIM 数据浏览，仍需要查看属性集、类型定义、材料等实体。

### 建议方案

- 为非几何实体提供独立数据浏览模式。
- 不参与三维高亮。
- 支持查看：
  - `IFCPROPERTYSET`
  - `IFCPROPERTYSINGLEVALUE`
  - `IFCMATERIAL*`
  - `*TYPE`
- 支持从构件属性跳转到关联的属性集/类型/材料。

### 验收标准

- 非几何实体与三维构件交互明确分离。
- 用户可以从构件追溯属性、类型和材料信息。

## 暂不纳入最小 MVP 的原因

这些优化会提升大模型体验，但不是最小 MVP 的前置条件。当前 MVP 应优先完成：

- 真实 `.frag` 文件加载。
- 模型渲染。
- 基础模型树。
- 构件选择与高亮。
- 构件属性展示。
- 基础视角控制。

上述能力已经进入基础实现阶段。本计划当前不再安排新的树功能，优先使用真实大模型验收分页、虚拟滚动、搜索、缓存命中和分组高亮阈值。
